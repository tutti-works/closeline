# アーキテクチャ

## 構成

- `src/components`: React UIとCanvas描画
- `src/game`: ゲーム状態、ルール、CPU
- `src/game/geometry`: 幾何計算、線分判定、面検出
- `src/game/rules`: 配置・得点・勝敗判定
- `src/hooks`: React向け状態フック
- `src/types`: 共有型

## 状態管理

`GameState` はプレイヤー、手番、線分一覧、獲得領域一覧、設定、勝敗モードを保持します。描画用の一時ドラフトはReactコンポーネント内だけに置き、確定済みゲームデータとは分離しています。

## 幾何計算

線分配置時には、端点を既存端点、既存線分、盤面外枠へ一定距離内でスナップします。その後、盤面外へのはみ出し、既存線との交差、重複、獲得済み領域への侵入を検証します。

領域検出は、確定済み線分と盤面外枠から平面グラフを作ります。既存線の途中に端点が接続された場合は、その接続点で辺を分割します。隣接辺を角度順に並べ、半辺をたどって閉じた面を抽出し、既存領域のシグネチャと比較して新規領域だけを獲得対象にします。

面積は多角形のshoelace formulaで計算します。単純なピクセル塗りつぶしではなく、ゲームデータとして再現可能な幾何情報を保存します。

## CPU

CPUは合法手候補を多数生成し、難易度に応じて評価します。評価には即時獲得面積、獲得領域数、中央からの距離、盤面展開のしやすさを含めています。完全探索ではありませんが、ランダム配置だけに偏らないようにしています。

## 保存形式

localStorageに `closeline.savedGame.v1` として `GameState` 全体をJSON保存します。ページ再読み込み後も途中から再開できます。
