# アーキテクチャ

## アプリ構成

CLOSELINEはReact + TypeScript + Viteで構成した静的Webアプリです。盤面はSVGで描画し、ゲーム内座標は常に `0..1` の正規化座標で扱います。

## 状態管理

`GameState` は線、交点ドット、三角形陣地、現在手番、設定を保持します。React stateで管理し、変更ごとにlocalStorageへ保存します。

保存キーは `closeline.v2.state` です。

## 幾何計算

`src/game/geometry` に線分交差、proper intersection、線分重複、点と線分の距離、縦線禁止、三角形面積、三角形内外判定、三角形同士の接触判定を分離しています。

## ドット管理

交点は `IntersectionNode` として管理します。現在のルールでは、交点を最初に作ったプレイヤーだけがそのドットを使えます。後から相手が同じ交点を通っても、相手の所有権は追加しません。

## 三角形検出

三角形検出は、現在プレイヤーが所有する有効ドット3点の組み合わせを調べます。各辺が実在する線分で連続的にカバーされている場合だけ三角形として成立します。既存陣地との重なりや接触は禁止します。

勝敗は獲得した三角形の面積合計で判定します。同点時は三角形数、最大三角形面積の順に比較します。

## CPU

CPUは完全探索ではなく、候補生成と評価関数による近似です。

候補生成では以下を混ぜています。

- 中立線、自分の線、相手の線の周辺候補
- 既存ドットから線を伸ばす候補
- 所有ドット同士を結ぶ候補
- 盤面の活動中心から外側へ展開する候補
- ランダム候補

HARDでは、即時に三角形を取れる手、自分の次ターンの三角形チャンス、三角形獲得による相手チャンスの先取りを戦術評価に入れています。

CPU判断ログは `closeline.v2.aiDecisionLogs` に保存します。ログには候補数、選択手、上位候補、獲得予定面積、交点数、評価スコアを保存し、スコアパネルから確認できます。

## AI学習

オフライン学習は `npm.cmd run train:ai` で実行します。TypeScriptのゲームルールをNode向けに一時バンドルし、自己対戦からCPU評価用の重みを更新します。結果は `src/game/cpu/learnedWeights.json` に保存され、ブラウザ側CPUが候補手評価に使用します。

推奨コマンド例:

```powershell
npm.cmd run train:ai -- --games=200 --turns=18 --lr=0.05 --workers=8 --progressEvery=20
```

主なオプション:

- `--games`: 自己対戦ゲーム数
- `--turns`: 1ゲームの最大ターン数
- `--lr`: 学習率。200 gamesなら `0.05`、1000 gamesなら `0.03` が目安
- `--workers`: Node.js `worker_threads` による並列自己対戦数。Core i7-13700Fではまず `8`、余裕があれば `12`
- `--progressEvery`: 進捗表示間隔

学習は既存の `learnedWeights.json` から継続します。自己対戦の先攻はプレイヤー/CPUで交互に切り替えます。学習時の候補生成には、軽量な訓練用候補に加えてブラウザCPUの `generateMoves` も混ぜ、実戦で選べる手と学習対象のズレを減らしています。

少数ゲームの学習結果はブレやすいため、実用確認は200 games以上を推奨します。

## デプロイ

サーバー処理のない静的アプリなので、Cloudflare Pagesなどの静的ホスティングに向いています。

Cloudflare Pagesの設定:

- Build command: `npm run build`
- Build output directory: `dist`

## 制限

- CPUは完全探索ではありません。
- AI学習は深層学習ではなく、評価重みの調整です。
- 候補生成に含まれない手は、学習しても選べません。
